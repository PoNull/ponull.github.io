搭建节点订阅转换服务器

无标签
发布日期:   2023-11-02

有时候用别人的订阅服务, 在更新节点的时候总感觉拉不动, 于是就想着自己搭建一个, 顺便学习一下

SubConverter后端搭建

源码地址: tindy2013/subconverter

下载并解压项目release 二进制文件

cd /root
wget https://github.com/tindy2013/subconverter/releases/download/v0.8.1/subconverter_linux64.tar.gz
tar -zxvf subconverter_linux64.tar.gz

修改配置文件参数

找到解压出来的目录的subconverter/pref.ini文件

这里会有三个文件pref.example.ini pref.example.toml pref.example.yml
好像都是一样的, 这里我们使用pref.example.yml进行修改，我觉得yml语法简单清晰一些，其它的也同理

cd /root/subconverter
cp pref.example.yml pref.yml
vim pref.yml
找到如下参数进行修改
common:
  api_access_token: subconvertertoken            #随意设置自己知道就行

managed_config:
  managed_config_prefix: https://sub.domain.com  #设置成我们刚刚解析的域名

server:
  listen: 127.0.0.1                              #这里改成 127.0.0.1 进行反代

创建服务进程并启动

接下来我们需要创建一个服务，让VPS每次重启或是开机自动运行后端程序

找到VPS目录 /etc/systemd/system，创建一个名为 sub.service 的文件

打开文件，贴入以下内容，保存。

[Unit]
Description=A API For Subscription Convert
After=network.target

[Service]
Type=simple
ExecStart=/root/subconverter/subconverter
WorkingDirectory=/root/subconverter
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

检查运行状态以及设置开机自启

systemctl daemon-reload
systemctl start sub
systemctl enable sub
systemctl status sub

到这里，后端就搭建完毕了，接下来我们需要配置反代

反代配置

安装nginx

apt install nginx

配置反代

vim /etc/nginx/sites-available/default
找到server节点，修改如下
server {
    listen 80 sub.domain.com;
    listen [::]:80 sub.domain.com;

    server_name _;

    location / {
        proxy_pass http://127.0.0.1:25500;
    }
}

前端搭建与配置

下载并安装 Sub-Web

git clone https://github.com/CareyWang/sub-web.git
cd sub-web
npm install # 或者 yarn install

运行程序

npm run serve # 或者 yarn serve

修改版

原项目 不支持node18了, 可以用以下项目代替
https://suburl.v1.mk

再次修改

我已经用react重写了, 直接去仓库查找即可 订阅接口使用, 基本上都是使用的这个项目接口https://suburl.v1.mk
